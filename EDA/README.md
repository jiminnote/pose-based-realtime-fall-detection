0. 데이터 구조 분석 및 EDA 요약
1. 클래스 분포 / 카메라 편향 / 시나리오 편향
클래스별 샘플 수, 카메라(C1~C8)별 분포, 시나리오 그룹별 분포를 집계하여 데이터의 불균형 및 편향 여부를 확인함.
클래스/카메라/시나리오별로 큰 편향 없이 비교적 고르게 분포되어 있음.
2. Video Metadata 분석 (FPS, 길이, 해상도)
영상의 해상도는 대부분 1280x720으로 통일됨.
영상 길이는 10초, 프레임 수는 100개 등 다양함.
영상 프레임 수와 FPS(초당 프레임 수)는 샘플마다 다를 수 있으나, 다운샘플링 후에는 10초 100프레임(10fps) 등으로 정규화됨.
3. Sensor Sequence EDA (길이, 분산, peak 존재 여부)
센서 시퀀스는 원본 기준  100개(10Hz, 10초) 구성됨.
센서 시계열의 분산, peak 등 이상치 여부를 시각화로 점검함.
다운샘플링 시 600→100개(6개씩 평균)로 영상 프레임과 1:1 매칭 가능.
4. Image 10-frame 품질 점검 (순서, 밝기, 누락 여부)
각 샘플마다 이미지 10장이 정상적으로 존재하는지, 순서/밝기/누락 여부를 확인함.
이미지 개수는 대부분 10개로 정상, 누락/이상치 거의 없음.
5. Multimodal alignment 검사 (센서 길이 vs 영상 길이)
센서 row 수와 영상 프레임 수가 일치하는지, 시간축 동기화가 잘 맞는지 확인함.
다운샘플링 후에는 센서와 영상이 1:1로 매칭되어 멀티모달 분석에 적합함.
6. 라벨 의미 검증 (SY가 실제 낙상 동작인지 샘플 확인)
SY(낙상) 클래스 샘플의 센서/영상/이미지 시퀀스를 시각화하여 실제 낙상 동작이 맞는지 육안 검증함.
라벨 오류나 이상치 샘플은 거의 없음.
결론:
데이터 구조와 품질이 전반적으로 양호하며, 멀티모달 정렬 및 라벨 신뢰도도 높음.
