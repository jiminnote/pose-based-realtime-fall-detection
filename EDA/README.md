	- N: 2272
	- SY: 1856

## 2. 클래스 분포 / 카메라 편향 / 시나리오 편향
- 클래스/카메라/시나리오 분포를 집계해 큰 편향은 없는지 확인
- 큰 편향 없이 비교적 고르게 분포(추가로 “카메라별 성능 분해”는 모델 단계에서 재확인 예정)

## 3. Video Metadata 분석 (FPS, 길이, 해상도)
- 라벨/가공 기준 목표: `scene_length=100` (10초를 100프레임으로 정규화)
- 실제 분포 예시(SY 라벨 검증 기준 1856개):
	- `video_length` 평균 100.62, 최소 100, 최대 244 (일부 outlier 존재)
- 참고: 일부 경로에서 비디오 메타 추출이 실패(`fps/frame_count/width/height/duration=None`)할 수 있어, 메타 수집 시 **파일 유효성 체크/예외 처리**가 필요.

## 4. Sensor Sequence EDA (길이, 분산, peak 존재 여부)
- 원본 센서 길이 분포(노트북 04): `seq_length`가 0 또는 600에 몰림(품질/파싱 이슈 가능)
현재 파라미터/채널 선택으로는 peak가 검출되지 않음(재설계 필요)
- 정규화 정책(프로젝트 표준): 센서도 영상과 동일하게 길이를 100으로 맞춤
	- SY 라벨 검증(1856개): `sensor_length`가 전부 100으로 정규화됨(평균/표준편차 100/0)

## 5. Image 10-frame 품질 점검 (순서, 밝기, 누락 여부)
- 각 샘플당 이미지 10장 존재 여부를 확인.
- 전체(4128개) 기준 `num_images` 평균/최소/최대 모두 10 → 누락/이상치 거의 없음.

## 6. Multimodal alignment 검사 (센서 길이 vs 영상 길이)
- 매칭된 샘플 수: 4128
- 센서-영상 길이 산점도/차이 분포에서,
	- 정규화된 케이스(100 vs 100)는 잘 맞음
- 센서 HZ 추정 분포(노트북 06): 값이 1, 166.67, 500 등으로 튀는 다봉 형태 → “row 수/시간 메타” 기반 추정치라 데이터마다 기준이 섞였을 가능성

## 7. 라벨 의미 검증 (SY가 실제 낙상 동작인지 샘플 확인)
- SY(낙상) 샘플의 센서/영상/이미지 시퀀스를 육안 검증.
- 라벨 오류나 이상치 샘플은 크게 보이지 않음

## 결론
- 멀티모달 데이터 구조는 일관적이며, 이미지(10장)는 매우 안정적.
- 현재 파이프라인의 핵심 표준은 “`scene_length=100`에 맞춘 영상/센서 정규화”이며, 대부분은 이를 만족.
- 남은 리스크는 (1) 일부 비디오 메타 수집 실패, (2) 센서/영상 길이 outlier(정규화 누락) 및 (3) 센서 Hz/peak 관련 지표 재정의.
